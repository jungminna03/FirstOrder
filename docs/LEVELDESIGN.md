## 1. 레벨 디자인 철학

난이도는 **3단계 체감 순서**로 플레이어에게 점진적으로 노출된다.

```
브릭 속도 증가 (0~180s)
  → 브릭 HP 증가 (90s~)
    → 브릭 밀도 증가 (웨이브 기반, ~180s 포화)
```

180s 이후에는 속도·밀도가 고정되고 **HP만 계속 오르는** 후반 엔드리스 구간으로 전환된다.

## 2. 시간 기반 난이도 레버

### 레버 1: 브릭 하강 속도

| 경과 시간 | 속도 (px/s) | 비고 |
|---|---|---|
| 0s | 50 | 시작 |
| 30s | 60 | |
| 60s | 70 | |
| 90s | 80 | |
| 120s | 90 | |
| 150s | 100 | |
| **180s** | **110** | 🔒 이후 고정 |

- 틱 주기: **30초마다 +10 px/s**
- 코드 상수: `DESCENT_SPEED` 초기값 (GameScene.js)

---

### 레버 2: 브릭 HP

| 경과 시간 | HP | 증분 |
|---|---|---|
| 0~90s | 1 | — |
| 90s | 3 | +2 |
| 120s | 7 | +4 |
| 150s | 13 | +6 |
| 180s | 21 | +8 |
| 210s | 31 | +10 |
| 240s | 43 | +12 |

- **공식:** `HP(n) = HP(n-1) + (2 × tick_index)`  (90s 이후)
- 속도·밀도 캡 이후에도 **엔드리스로 계속 증가**하는 유일한 레버.

---

### 레버 3: 브릭 행 밀도

```
density = clamp(0.08 + (waveCount - 1) × 0.012, 0.08, 1.0)
```

| 웨이브 | 밀도 | 브릭 수 (10열 기준) |
|---|---|---|
| 1 | 0.08 | ~1개 |
| 20 | ~0.31 | ~3개 |
| 40 | ~0.55 | ~5~6개 |
| 77 | 1.0 | 10개 (꽉 찬 행) |

- 웨이브당 **최소 1개** 보장
- **웨이브 77 (≈180s)** 이후 밀도 1.0 고정

## 4. EXP & 레벨 스케일링

### EXP 획득
- 브릭 파괴 1회당: `EXP_PER_KILL = 10`
- `expBoost` 퍽 적용 시: ×1.3 (최대 3중첩 → ×1.9)

### 레벨업 EXP 임계값

```
EXP_TO_NEXT(level) = EXP_BASE × 1.4^(level - 1)
EXP_BASE = 100
```

| 레벨 | 필요 EXP | 누적 EXP |
|---|---|---|
| 1 → 2 | 100 | 100 |
| 2 → 3 | 140 | 240 |
| 3 → 4 | 196 | 436 |
| 4 → 5 | 274 | 710 |
| 5 → 6 | 384 | 1094 |
| 6 → 7 | 538 | 1632 |
| 7 → 8 | 753 | 2385 |
| 8 → 9 | 1054 | 3439 |

## 7. 난이도 곡선 요약 (디자인 노트)

```
[쉬움]  0~90s   : 속도만 증가. 볼 1개로 충분히 처리 가능.
[중간]  90~180s : HP 급등 시작. 퍽 시너지가 필수화됨.
[어려움] 180s~  : 속도·밀도 고정, HP만 폭증. 고데미지 빌드 필수.
```

### 플레이어 대응 패턴
- **초반 (0~90s):** 어떤 퍽이든 무난. `magnetBar`, `overdrive` 선호.
- **중반 (90~180s):** `heavyMetal`, `momentum`, `bombShot` 등 데미지 빌드 필요.
- **후반 (180s~):** `hydraBar` + `novaBurst` / `timeStop` 조합이 아니면 점점 불리.
